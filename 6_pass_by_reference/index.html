6. 値渡しと参照渡し

プリミティブ型

プリミティブ型に該当するのは、数値、文字列、ブーリアン、undefined、null.

・プリミティブ型の値を代入した場合、

数値10が保管され、その場所が変数aに格納される.
let a = 10;
変数 a P1
→
10
aに20を代入すると、20が新しく別の場所に保管され、その場所がaに格納される.
a = 20;
変数 a P2
→
10
20

・複数の変数に同じ値を代入した場合、

値が複数の場所に保管され、aとbにはそれぞれ別の場所が格納される.
let a = 10;
let b = 10;
変数 a P1
変数 b P2
→
→
10
10

・変数に別の変数を代入する場合

aに保管場所が格納された後、bにはaと同じ場所が格納される.
let a = 10;
let b = a;
変数 a P1
変数 b P1
→
↗
10
bだけ変更すると
新しい保管場所が作られ、新しい場所がbに格納される.
b = 20;
変数 a P1
変数 b P2
→
→
10
s
20
プリミティブ型の値を代入する場合は、一時的に複数の変数が同じ値を参照することはあるが、変数に対する操作が別の変数が参照している値に影響を与えることはない.
これを値渡しという.

オブジェクト型

オブジェクト型とはプリミティブ型以外のもので、配列やオブジェクト、関数など.

・変数に値を代入した後で別の値を代入する場合

let a = [10, 20];
配列[10,20]が保管され、その場所が変数aに格納される.
変数 a P1
→
[10, 20]
a = 20;
新しく[30, 40]を代入すると、新しく別の場所に保管され、その場所がaに格納される.
変数 a P2
→
[10, 20]
[30, 40]
a[0] = 50;
aが参照している配列に対してインデックス0の要素を50に置き換える.
この場合、aに格納されている場所は変化しない.
変数 a P2
→
[10, 20]
[50, 40]

・変数に別の変数を代入する場合

let a, b;
a = [10, 20];
b = a;
配列[10,20]が保管され、その場所が変数aに格納される.
次に変数aに格納されている場所が変数bに格納される.
変数 a P1
変数 b P1
→
↗
[10, 20]
b[0] = 50;
bが参照している配列に対してインデックス0の要素を50に置き換える.
この場合、aおよびbに格納されている場所は変化しない.
つまり、変数bに対する操作が変数aが参照している値にも影響する.
変数 a P1
変数 b P1
→
↗
[50, 20]
オブジェクト型の値を変数に代入する場合、一時的に複数の変数が同じ値を参照することがある点はプリミティブ型の場合と同じ.
その時に例えば変数が参照している配列の要素を置き換えるなどの操作を行うと、変数に対する操作が別の変数が参照している値に影響を与えることになる.
これを参照渡しという.
